## Decisions

1. **Tech stack** — Mirror MF app: React 19 + TypeScript + Vite 6 + TanStack Router + Dexie (IndexedDB) + Tailwind CSS 4 + shadcn/ui + Recharts. [Rationale: proven architecture, user is familiar, no backend needed.]
2. **Stock scoring** — 17 parameters from study material, scored/weighted per-parameter, composite score out of 100. Modeled after MF app's 23-factor scoring engine.
3. **Data sourcing** — Two-tier approach: (a) `nse-bse-api` npm package for live quotes and corporate actions, (b) Screener.in scraping for fundamental data (P/E, ROE, ROCE, promoter holding, etc.). Fundamentals cached in IndexedDB with daily refresh. Manual override/entry available as fallback.
4. **Dark mode** — Included at launch (same pattern as MF app).

## 2026-06-08 — Revised Data Sourcing Strategy

- **Context:** User rejected the assumption that fundamentals would be entered manually. Data must be scraped from the web.
- **Decision:** Use Screener.in (`screener.in/company/{slug}/`) as the primary source for fundamental data. Scraping runs client-side via CORS proxy (same Vite/nginx pattern as MF app's API proxies). nse-bse-api continues to handle live quotes.
- **Risk:** Screener.in may change HTML structure or implement anti-scraping measures. Mitigation: isolated scraping module, cache-first strategy, manual override fallback.
- **Cost:** Zero — Screener.in is free to use.
